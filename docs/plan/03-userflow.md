# 유저 플로우 — 오제로의 거울

## 메인 플로우 — 하루 루프 (순서 변경 금지 · 제출 후 뒤로 못 감)

```mermaid
flowchart TD
  H[S01 홈: 21일 격자] -->|서버가 '열림' 판정| O[S02 오프닝: 어제의 회수]
  H -->|아직 안 열림| W["'오늘 기록은 HH시에 열려요'"]
  O --> C[S03 상태 체크: 숫자 4종]
  C --> F[S04 자유 기록]
  F --> K{위기 신호?}
  K -->|정상| U[S06 사용자가 먼저 구별]
  K -->|감지| L1[하단에 조용한 한 줄 표시 — S05] --> U
  U --> M[S07 AI 거울 대조 — 나란히]
  M --> L[S08 망상·감정 연결]
  L --> Q[S09 오늘의 질문 → 답변 + 공개 토글]
  Q --> A[S10 작은 행동 선택 + 낮 알림 스위치]
  A --> D[S11 제출 완료: N명 카운트 + 익명 답변 1개]
  D --> H
```

핵심 순서 규칙: **사용자 구별(S06)이 AI 대조(S07)보다 반드시 먼저.** AI가 먼저 답을 주면 훈련이 아니라 의존이 된다. 타인의 답변(S11)은 제출 전에는 어떤 형태로도 노출되지 않는다.

## 재방문 플로우 — 빠진 날 (실패가 아니라 데이터)

```mermaid
flowchart TD
  P[푸시: 고정 시각 알림] --> H[S01 홈]
  H --> O{어제 기록 있나?}
  O -->|있음| Y["오프닝: 어제 문장 인용 + 행동 실행 여부"]
  O -->|없음| N["오프닝: '어제는 기록이 없는 날입니다. 안 쓴 날은 어떤 날이었나요.'"]
  N --> C[상태 체크부터 정상 진행]
  Y --> C
```

- 연속 기록이 깨졌다는 표시·문구 일절 금지.
- 3일 연속 미기록이면 저녁 푸시의 문구만 바뀐다 (추가 발송 없음): "3일째 기록이 없습니다. 안 쓴 날들도 데이터입니다."
