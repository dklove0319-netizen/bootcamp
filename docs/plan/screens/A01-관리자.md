# A01. 관리자 대시보드 (운영자 1인 전용)

## 들어오는 길
- 관리자 계정(is_admin=true)으로 로그인 → 별도 주소(/admin). 일반 사용자에게 링크 비노출 + 서버에서 권한 확인 (링크 숨김만으로는 보호 아님)

## 화면 구성 (위에서 아래로)
1. 사용자 수동 등록: 이메일 입력 + 익명 코드 칸 (선택) → 등록 (결제 전 운영 방식)
   - 코드 칸을 비우면: 번호표 기계에서 자동 발급 (o056부터 — 레터에서 o055까지 발급됨)
   - 기존 수료자면: 원래 코드(예: o023)를 입력 → 그대로 사용 (중복이면 등록 실패)
2. 사용자 목록: 코드·시작일·최근 기록일·진행 일차 (한 번에 모아 읽기 — N+1 금지)
3. 사용자별 기록 열람 (전체 필드 — 열람 사실은 약관·동의에 명시된 구조)
4. 이탈 지표: 날짜별 이탈(며칠째에 끊기나) · 단계별 이탈(어느 단계에서 나가나) · 답변 길이 추세

## 규칙 (지시서 7번)
- 운영자 1인 전제. 관리자 권한 부여는 DB에서 수동 (화면에 권한 관리 기능 안 만듦 — 최소 코드).
- 기록 수정·삭제 기능 없음 — 열람만 (사용자 데이터의 주인은 사용자).

## 행동과 흐름
| 행동 | 성공하면 | 실패하면 |
|---|---|---|
| 수동 등록 | "등록했어요 (코드: oNNN)" + 목록 갱신 | 이메일 중복: "이미 등록된 이메일이에요" / 코드 중복: "이미 쓰이는 코드예요" |
| 기록 열람 | 상세 표시 | "불러오지 못했어요" + 재시도 |

## 예외 5종
| 상황 | 처리 |
|---|---|
| 비어 있을 때 (사용자 0명) | "등록된 관찰자가 없어요" + 등록 칸 |
| 로딩 중 | 표 스켈레톤 |
| 실패 | 문구 + 재시도 |
| 권한 없음 (일반 사용자가 /admin 접근) | 404 처리 (관리자 페이지의 존재 자체를 안 알림) |
| 잘못된 입력 | 이메일 형식 오류 문구 |

## 읽고 쓰는 데이터

- 쓰기: 사용자 등록 (Auth 초대 + profiles)
