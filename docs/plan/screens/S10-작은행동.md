# S10. 작은 행동 선택 (하루 루프 단계 10 — 행동의 주어는 사용자)

## 들어오는 길
- S09 답변 제출 → 자동 진행

## 화면 구성 (위에서 아래로)
1. 안내 한 줄: "내일 할 작은 행동을 하나 정해요."
2. 행동 입력칸 (한 줄)
3. "AI 제안 보기" 버튼 — 누르기 전에는 제안이 어떤 형태로도 안 보임 (버튼 뒤에 숨김)
4. 낮 알림 스위치: "낮에 이 행동을 알림으로 받을까요" — 기본 꺼짐
5. 오늘 기록 마치기 버튼

## 규칙 (지시서 2-5, 3-10, 5번)
- 행동을 정하는 것은 사용자. AI 제안은 명시적 요청(버튼) 시에만.
- 스위치를 켜면 다음날 낮 리마인더 발송 — 문구는 이 칸의 원문 인용: "어제 당신이 정한 행동 — '(원문)'"
- 마치기를 누르면 submitted_at 기록 → 오늘 루프 종료, 수정 완전 불가 (오타 포함 — 결정 2026-07-05).

## 행동과 흐름
| 행동 | 성공하면 | 실패하면 |
|---|---|---|
| AI 제안 보기 | 제안 1~2개 표시 (선택은 자유 — 채택 강제 없음) | "제안을 가져오지 못했어요" (직접 쓰기는 계속 가능) |
| 오늘 기록 마치기 | action_text·action_reminder·submitted_at 저장 → S11 | "저장하지 못했어요. 다시 시도해주세요." (입력 보존) |

## 예외 5종
| 상황 | 처리 |
|---|---|
| 비어 있을 때 (행동 미입력) | 마치기 누르면: "행동을 하나 적어주세요" — 정하지 않기 허용 여부는 빌드 때 확인 [기본: 필수] |
| 로딩 중 | 저장 중 버튼 비활성 |
| 실패 | 문구 + 재시도 |
| 권한 없음 | S14로 |
| 잘못된 입력 | 빈 입력 외 제한 없음 |
