# S17. 무료 측정 — 즉시 측정 흐름 (스펙 1단계 · 비로그인 가능)

> 근거: docs/PRICING_SPEC.md — 기록 1건 → 분석 1회. 비로그인 1일 1회, 로그인 1일 3회. 결과 저장만 가입 필요.
> 용어: 전 화면 "망상"으로 통일 (2026-07-05 사용자 확정 — 이층 가정 폐기, 06-decisions.md).

## 들어오는 길
- 홈(측정 입구) · 공유 카드의 링크 (블럭 6)

## 흐름 (화면 3개가 한 줄로)
1. **입구**: 서비스명 + 한 줄 소개 + "지금 비춰보기" 버튼 (가격·가입 요구 없음)
2. **기록 입력**: 쓰기 가이드(고정 문안 — S04와 동일 계열, 첫 문장: "잘 쓸 필요 없어요") + 여러 줄 입력칸 + 측정 버튼
3. **결과**: 스펙 절대 규칙 1 형식 그대로 —
   - 사용자 기록 직접 인용 (원문 그대로 — 서버 일치 검증, 창작·각색 금지)
   - 사실/망상 측정값 (예: "사실 2 : 망상 7")
   - 미완결 질문 하나로 끝남
   - 유형·성격 라벨 금지. 위로·조언 금지
   - 하단: "결과를 저장하려면 가입이 필요해요" (블럭 3) + 21일 코스 안내 한 줄 (블럭 10에서 가격 노출)

## 규칙
- AI 프롬프트는 서버에만. 비로그인 기록은 저장하지 않는다 (저장은 가입자만 — 스펙).
- 횟수 제한은 서버 판정: 비로그인 1일 1회, 로그인 1일 3회. 초과 시: "오늘 측정은 여기까지예요. 내일 다시 비춰볼 수 있어요."
- 이 측정은 맛보기(거울을 한 번 보여주는 것)다 — 21일 코스의 "사용자 먼저 구별" 훈련 순서는 코스 안에서만 적용된다.

## 행동과 흐름
| 행동 | 성공하면 | 실패하면 |
|---|---|---|
| 지금 비춰보기 | 기록 입력 화면 | — |
| 측정 | AI 판독 → 결과 화면 | "측정하지 못했어요. 다시 시도해주세요." (작성 내용 보존) |
| (초과 시) 측정 | — | 위 제한 안내 문구 |

## 예외 5종
| 상황 | 처리 |
|---|---|
| 비어 있을 때 (기록 미작성) | 측정 누르면: "한 줄이라도 적어주세요" |
| 로딩 중 | "비추고 있어요" + 버튼 비활성 (수 초) |
| 실패 (AI·통신) | 문구 + 재시도, 입력 보존. 인용 검증 실패 성분은 버리고 나머지만 표시 |
| 권한 없음 | 해당 없음 (비로그인 허용 화면) |
| 잘못된 입력 | 빈 입력 외 제한 없음 |

## 읽고 쓰는 데이터
- 비로그인: 저장 없음 (일시 처리). 횟수 제한용 최소 식별만 서버에서
- 로그인: 가입자는 측정 결과 저장 (블럭 3에서 정의)
