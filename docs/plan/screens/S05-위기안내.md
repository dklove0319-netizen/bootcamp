# S05. 위기 — 조용한 한 줄 (B안 "믿음 + 비상구" · 독립 화면 아님, 구성 요소)

> 2026-07-05 결정 (06-decisions.md): 지시서 원안(루프 중단 + 안내 화면 전환)을 변경.
> 오제로는 사용자를 환자로 보지 않는다 — 루프는 절대 멈추지 않는다. 대신 비상구 정보 하나가 조용히 그 자리에 있는다.

## 언제 나타나나
- S04 자유 기록 저장 직후 서버 위기 판정(키워드 1차 + AI 2차)이 crisis일 때
- 그날 루프의 이후 화면들(S06~S11) 하단에 고정 한 줄로 표시. 루프 진행은 평소와 완전히 동일

## 형태 (화면 하단 한 줄 · 닫기 가능)
- 고정 문안: "이야기할 수 있는 곳이 있어요 — 자살예방 상담전화 109 (24시간)." 탭하면 전화(tel: 링크)
- 닫기(x)를 누르면 그날은 다시 표시하지 않음
- AI 생성 문구 아님 — 고정 문안만. 위로·질문·평가 없음

## 규칙
- 진단·중단·"괜찮으세요?" 금지. 정보 제시뿐.
- 위기 내용의 판독(구별·대조·질문 생성)은 다른 기록과 완전히 동일하게 — 특별 취급이 곧 환자 취급이다.
- crisis_detected 저장. 관리자에게 별도 통지 없음 — 대시보드 열람 시 표시만 (프라이버시 우선).
- 다음날은 아무 흔적 없이 정상 재개.
- AI 판정 실패(장애) 시 키워드 1차만으로 판정하고 진행 — 판정 때문에 루프를 세우지 않는다.
- 설정(S15)에 같은 연락처를 상시 항목으로 둔다 (트리거 없이도 항상 접근 가능).
- **공개 확장 전 재검토 항목**: 첫 사용자(수료자·수동 등록·성인)를 넘어 모르는 대중에게 열기 전, 이 수위가 충분한지 security-reviewer 점검과 함께 다시 판단한다.

## 예외 5종
| 상황 | 처리 |
|---|---|
| 비어 있을 때 | 해당 없음 (조건 표시 요소) |
| 로딩 중 | 해당 없음 (정적 한 줄) |
| 실패 | 판정 실패 시 키워드 1차 결과만 사용 (위 규칙) |
| 권한 없음 | 해당 없음 (루프 안에서만) |
| 잘못된 입력 | 해당 없음 |
