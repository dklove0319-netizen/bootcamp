# S04. 오늘의 기록 — 자유 기록 (하루 루프 단계 3)

## 들어오는 길
- S03 상태 체크 → 다음

## 화면 구성 (위에서 아래로)
1. 쓰기 가이드 (입력칸 바로 위, 매일 고정 — 문구 확정 2026-07-05):
   > 잘 쓸 필요 없어요. 문장이 아니어도 돼요.
   > 떠오르는 대로 쓰세요. 욕이 나오면 욕을 쓰세요.
   > 고치지 말고, 검열하지 말고, 배설하듯 쏟아내세요.
   > 맞는 글, 틀린 글은 없어요. 이 기록은 다른 관찰자에게 보이지 않아요.
2. 자유 기록 입력칸 (여러 줄, 화면 대부분 차지 — 모바일에서 쓰기 편하게)
3. 다음 버튼

## 규칙
- **내용을 유도하는 문구(주제·예시 문장)는 금지, 형식의 허가(잘 쓸 필요 없음·검열 금지)는 매일 보여준다** — 가이드는 "무엇을 쓸지"가 아니라 "어떻게 써도 되는지"만 말한다. 무의식이 드러나려면 검열 문턱부터 내려야 한다.
- 가이드 문구는 21일 내내 동일 (눈금과 같은 원리 — 조건이 같아야 기록이 비교 가능).
- 마지막 줄은 사실이어야 한다: 자유 기록은 격리 층 — 다른 관찰자에게 절대 비노출 (본인+관리자만, 관리자 열람은 약관 명시). "당신만 봐요" 같은 부정확한 약속은 쓰지 않는다.
- localStorage 금지 — 작성 중 이탈 대비가 필요하면 서버 임시 저장으로 (단, 최소 구현 우선: v1은 제출 시 1회 저장, 이탈 시 유실 허용 여부는 빌드 중 확인).
- 다음을 누르면: 저장 → 위기 신호 판정(단계 4, 서버 — 키워드 1차 + AI 2차) → **어느 쪽이든 S06으로 정상 진행** (B안: 루프 무중단). 감지 시에만 이후 화면 하단에 조용한 한 줄 표시 (S05 참조).
- 이후 이 기록은 수정 완전 불가 — 오타 포함 (결정 2026-07-05, 기록의 원본성).

## 행동과 흐름
| 행동 | 성공하면 | 실패하면 |
|---|---|---|
| 다음 | free_text 저장 + 위기 판정 → S06 (감지 시 하단 한 줄 동반) | "저장하지 못했어요. 작성한 내용은 그대로 있어요. 다시 시도해주세요." |
| 위기 판정 자체가 실패(AI 장애) | 키워드 1차 결과만으로 진행 — 판정 때문에 루프를 세우지 않는다 | — |
