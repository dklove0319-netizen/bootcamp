# 운영자 입력 가이드 — 거울을 기획 의도에 맞춰가는 4가지 입력 (2026-07-10)

> "와, 정확히 나를 본다"의 카타르시스는 거울의 정밀도에서 나오고, 정밀도의 상한은 운영자의 눈이 정한다.
> 운영자가 주는 입력은 4가지뿐이며, 전부 이 창에 붙여넣기로 끝난다. 나머지(증류·주입·검증·배포)는 AI 몫.

## 입력 4가지

| # | 입력 | 방법 | 효과 | 실증 (2026-07-10 주간) |
|---|---|---|---|---|
| 1 | **원자료 (재료)** | 기록→레터 쌍을 원문 그대로 붙여넣기: "ozero26 / 5일차 기록: … / 그날 레터: …". 정리·요약 금지 | 가장 큼 | 해부 글 12편 → 채점 9/12→12/12 |
| 2 | **채점 (시식)** | AI가 만든 A/B 블라인드 시험지에서 고르기 + 한 줄 코멘트 | 시간 대비 최고 (10분 → 규칙 6개) | 블라인드 10장면 → 보정 6규칙 → 기각 4장면 재생성 |
| 3 | **현장 교정 (틀린 순간 잡기)** | 이상한 출력을 복사해 "이거 이상한데?"만 붙여넣기 (이유 몰라도 됨 — AI가 기준 대조로 원인 제시, 운영자는 확인만) | 즉시 · 1건 = 규칙 1개 | 세탁 사례 → 규칙 2개 / 아첨 사례 → 규칙 1개, 전부 회귀 문제집에 추가 |
| 4 | **규정 문서 (주방 규칙)** | 원칙이 정리되면 문서로 (자세규정 모듈처럼) | 가장 넓게, 가장 드물게 | 어휘+2겹 모듈 → 전 창구 당일 주입 |

## 운영자가 안 해도 되는 것
- 코드·기술 용어·구현 고민 (AI 몫)
- 자료 정리·요약 (원문 그대로가 더 좋다 — 요약하면 감각이 깎인다)
- "가르치는 법" 공부 (평소처럼 판정하면 AI가 레시피로 옮긴다)

## 붙여넣은 뒤의 파이프라인 (AI 몫)
금고 보관(docs/corpus, 깃 제외) → 증류(규칙·모범례) → 대본 주입(craft.ts) → 채점(운영자 시식 또는 자동 문제집) → 회귀 검사(기존 통과 문제 재시험) → 배포.

## 지금 순서 (2026-07-10 기준)
1. 기록→레터 쌍 붙여넣기 시작 (한 사람 분씩, 오는 대로)
2. 테스트 결제 후 운영자가 직접 21일 루프 돌기 — 매일 출력을 보면 입력 3이 저절로 돈다 (저녁 7시 알림 확인 겸)
3. 자료가 쌓이면 확장 채점표 30~50장면 (AI가 제작)
4. 심화 35 자료는 퍼널 열린 뒤
