# 변경 제안서 — 무료 측정을 "사실 칸 다시 쓰기 + AI 검증" 흐름으로 교체

> 상태: **방향 승인됨** (2026-07-07 사용자 지시 — 화면 구성까지 직접 설계). 즉시 구현.

## ① 뭘 바꾸나 (한 줄)
조각 태깅(기계 분할 + 사실/망상 버튼) 방식을 버리고, [오늘 기록 전부 쓰기 → 같은 화면에서 스크롤 → 사실·망상 기준 안내문 → 사실만 다시 쓰는 칸 → 다음 화면에서 AI가 사실 칸의 오염을 성분 단위로 짚어주기]로 교체한다.

## ② 왜
- 사용자 설계 지시. 자세규정 8장(사실 검증 모듈)의 화면화 — "이건 사실인가 해석인가"를 참가자의 실제 문장 위에서 반복시키는 핵심 훈련 장면.
- 기존 태깅 방식의 구조적 문제 해결: 기계 조각내기가 한국어 인용 구조를 찢고("귀찮나?" / "라는 생각이…" 분리), "싶었다." 같은 무의미 조각을 만들었다. 다시 쓰기 방식은 조각내기 자체가 필요 없다.
- 판별 기준 업그레이드 동시 반영: 구체성 검사("좋은 호텔"→망상, "리츠칼튼"→사실 — 평가어는 덧씌운 고정관념, 사실은 구체 이름·숫자) + 생각의 발생/내용 분리(내용 우선 판별 — "~라는 생각이 들었다" 세탁 구멍 봉쇄).

## ③ 영향
- 화면: /measure 2페이지로 (쓰기+기준+사실칸 → 결과). 태깅 화면 제거.
- 서버: /api/measure 입력 {text(원기록), facts(사실 칸)} — AI는 사실 칸을 성분 검사, 오염 1~2곳만 이유(자세규정 8-6: 전부 교정하면 훈계), 깨끗하면 조용히 통과(칭찬 금지).
- 저장: measurement.userSplit = 사실 칸 원문(facts), aiSplit = 성분 검사 결과. 표 구조 변경 없음(jsonb).
- 문서: 08-ai-prompts(기준 추가), CLAUDE.md(구체성 기준 한 줄), S17 개정.
- 횟수 제한·입구 사전 안내·영어 자동 전환은 그대로.
