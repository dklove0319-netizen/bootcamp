- 제안일: 2026-07-08 · 상태: **승인됨 (사용자 2026-07-08 "너의 추천대로 해줘") → E-2·E-3 구현 완료, E-4 대기**
- 출처: 사용자 제공 설계 문서 2편 ("7가지 장치" + "반영형 장치 5개와 바넘 경고") — 원문은 아래 '정본 보존'에.

## ① 뭘 바꾸나 (한 줄)

거울이 "오늘 하루"만 비추던 것을 넘어, **과거 기록 전체에서 같은 해석 구조를 찾아 원문 인용으로 반사하고, 사용자의 보정(0~10)으로 점점 정밀해지는 엔진**을 넣는다.

## ② 왜

- 카타르시스("맞아, 어떻게 알았어?")의 진짜 원천은 요약·공감이 아니라 **시간축 반복의 명명** — 연애·가족·일로 흩어진 사건들이 같은 구조였음을 보여주는 것. 이건 날짜에 흩어져 있어 사용자 자신은 못 보고, 데이터를 가진 앱만 할 수 있다 (실제 정보 비대칭 — 바넘·콜드리딩이 아님).
- 지금도 부품의 70%는 있다: 원문 저장(절대 규칙 2), 사실/망상 분리, 사흘의 거울(반복 인용+날짜), 인용 서버 검증, 절제 톤. **없는 것은 ③구조 매칭의 깊이 ④핵심 문장 공식 ⑤보정 루프**다.

## ③ 어떻게 구현하나 (파인튜닝 아님 — 4부품 조립)

AI는 "학습시키는" 게 아니라 매번 **편지**를 보내는 것: [규칙서] + [그 사람의 과거 원문] + [돌아온 답 검사] + [보정을 창고에 적어 다음 편지에 동봉]. "AI가 나를 기억한다"는 느낌은 매번 우리가 창고(DB)에서 기억을 꺼내 들려주기 때문에 생긴다.

| 사용자 문서의 장치 | 지금 상태 | 만들 것 |
|---|---|---|
| 1. 원문 보존 | ✅ 원문 그대로 저장 + 인용 서버 대조 | 해석 성분에 '계열 꼬리표'(예: 밀림/버려짐/부족함) 컬럼 추가 — 단, 사용자 노출 금지, 매칭 재료로만 |
| 2. 같은 구조 매칭 | 🔶 사흘의 거울(최근 3~14일, 문자 그대로의 반복만) | 질문 단계·중간 거울에서 **전체 기록 원문(≤21일)을 통째로 AI에 주고** 사건은 달라도 구조가 같은 것을 찾게 함. 21일 분량이면 프롬프트 창에 다 들어가므로 벡터DB 불필요 (v1 금지 원칙과 충돌 없음) |
| 3. 사실/해석/감정/행동 분리 | ✅ 세 칸 + 성분 판독 + 감정 연결 | 그대로 재활용 |
| 4. 보호 전략 해석 | ❌ 없음 | **가설형으로만** ("~기 때문일 수 있어요") + 반드시 보정 질문과 세트. 단정하면 마음 읽기(우리가 사용자 기록에서 잡아내는 바로 그 오염)가 된다 |
| 5. 한 문장 명명 | ❌ 없음 | 핵심 문장 공식: **"당신은 [사건] 때문에 무너지는 게 아니라, 그 사건을 [원문에서 반복된 해석]의 증거로 읽는 순간 무너집니다."** — 사람이 아니라 읽는 구조에 명명 |
| 6. 근거 제시 | ✅ 날짜+원문 인용 형식 존재 | 반사 문장마다 근거 인용 2~3개(날짜+원문) 필수 + 서버 검증 — 인용 못 대면 그 반사는 버린다 |
| 7. 보정 질문 | ❌ 없음 | 반사 뒤 "이 문장이 0~10 중 몇 점인가요? 더 가까운 말은?(무시/밀림/버려짐)" → `pattern_map` 표에 저장 → 다음 반사 때 규칙서에 동봉 ("사용자가 확인한 표현: 버려짐 8점") |

**엔진 흐름**: 기록 → 성분 분리(있음) → 해석 계열 꼬리표(신규) → 과거 원문과 매칭(신규) → 공식 문장+근거 인용 반사(신규) → 0~10 보정(신규) → pattern_map 갱신(신규) → 다음 반사에 주입.

## ④ 원칙 충돌 3곳 — 이렇게 지킨다 (두 설계 문서끼리도 충돌함)

1. **"당신은 ~한 사람" 예시 문장들** (첫 문서의 장치 5 예시) — 둘째 문서와 기존 절대 원칙이 둘 다 금지. **둘째 문서를 따른다**: 사람에게가 아니라 반복되는 해석·읽는 구조에만 이름 붙인다. 공식 문장은 허용, "~한 사람입니다"는 금지.
2. **숨은 믿음 추론** — "쓰지 않은 것을 지어내지 않는다"와 긴장. 해결: 믿음은 단정하지 않고 **인용 위에 세운 질문**으로만 ("세 날의 기록에 '밀렸다'가 반복돼요. 이 해석이 맞다면, 그 아래에 어떤 문장이 깔려 있나요"). AI가 문장을 제안할 땐 반드시 보정 질문과 세트.
3. **"정확해"를 표적 삼기** — 둘째 문서의 경고 그대로: 목표는 반응이 아니라 **데이터의 정확한 반영**. 반응을 노리는 순간 바넘으로 미끄러진다. 프롬프트에 명시하고, 검증 기준도 "사용자가 감탄했나"가 아니라 "인용이 전부 진짜인가"로 둔다.
