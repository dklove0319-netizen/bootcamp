# 변경 제안서 — 원자료 증류로 거울 정밀도 올리기 (증류 → 대본 주입 → 블라인드 채점)

## ① 뭘 바꾸나 (한 줄)

운영자가 제공한 원자료(추적 기록 7명 + 운영자 해부 글 12편, docs/corpus 금고)에서 **경계 규칙·각도 지도·선택 규칙·모범례**를 뽑아 AI 대본 4곳(구별 대조·오늘의 질문·중간 거울·보고서)에 주입하고, **블라인드 채점**으로 운영자와의 일치율을 측정해 1회 조정한다.

## ② 왜

- 지금 AI는 규칙(카메라 검사·구체성)만 알고 **실전 경계 사례와 각도 선택 감각**이 없다. 원자료에는 그 정답지가 통째로 있다: 사용자가 사실이라 쓴 문장 + 운영자의 해부(모범답안), 그리고 "상황 → 그날의 질문 → 다음날 반응(명중/미끄러짐)"의 짝.
- 운영자 확인(2026-07-10): 실력의 본체는 질문 목록이 아니라 **"이 사람의 오늘 자리를 읽고 어떤 각도가 지금 찌르는지 고르는 능력"** — 이것은 각도 지도 + 선택 규칙 + 명중례로 가르치고, 채점으로 검증한다.
- 이 제품의 재미 = 명료해지는 카타르시스 = 거울의 정밀도(기획 확정 2026-07-06). 이 작업이 곧 재미를 올리는 작업이다.

## ③ 영향 (어느 창구가 바뀌나)

**새 파일 1개**: `apps/web/lib/craft.ts` — 증류물을 상수로 한 곳에 관리 (대본 여기저기 하드코딩 금지)

**대본 주입 4곳** (화면·문구·표 구조 무변경, 시스템 프롬프트만):
1. `api/today/contrast` (하루 루프 6단계 구별 대조) + `api/measure` (무료 측정 구별) — 경계 규칙 + 구별 모범례
2. `api/today/question` (오늘의 질문 S09) — 각도 지도 + 선택 규칙 + 명중례
3. `api/observer/mirror3` (7·14일 중간 거울) — 선택 규칙(단계 감각)
4. `api/report` (21일 보고서 마지막 질문) — 각도 지도

**증류물 내용 (요약 — 전문은 craft.ts 에서 검토)**

경계 규칙 7 (해부 글 원전):
1. 수치 빠진 형용사·양 표현("가득·많다·사이 좋은") = 망상 → 개수·이름으로
2. 의도·결심("하려고 했다/포기했다") = 망상 — 카메라는 행동만
3. 타인의 인지 상태("까먹는다") = 망상 → 관찰된 행동으로
4. 개념어 포장(갈등·지적·자랑·소통·충돌) = 망상 → 실제 발화 문장·물리 행동으로
5. 부재의 진술("~한 일은 없었다") = 사실 아님 → 실제 있었던 행동·대화만
6. 긍정·성찰 문장도 동일 기준 (교정·경고 톤 금지 유지)
7. 해석이 크게 붙은 자리 신호론 — 감정이 클수록 그 해석 안에 핵심 믿음 (내부 참고용, 사용자 노출 금지)

각도 지도 (레터 커리큘럼에서 복원 — 질문이 겨누는 자리 5종):
증명 가능성 → 믿어서 얻는 것 → 숨은 욕구 → 자기상("나는 ~한 사람") → 회피 중인 행동·선택

선택 규칙 6 (기록에서 실측된 것만):
- 같은 해석이 다른 날짜 2회 이상 → 그 문장을 원문 인용으로 겨눈다 (기존 E-2 유지·강화)
- 알아차림-정체 신호("아차 싶긴 한데 그대로") → 더 파지 않고 몸 감각·행동 각도로
- 감정 어휘가 유독 큰 조각 → 그 해석을 겨눈다
- 압도·위기 신호 → 찌르지 않는다. 안전 확보 우선 (기존 위기 분기 유지)
- 초반(1~7일)은 증명·구별 각도, 중반(8~14) 욕구·이유, 후반(15~21) 자기상·선택 — 레터 커리큘럼의 단계 감각
- 과부하 신호(바쁨·번아웃 언급) → 가벼운 각도, 짧게

모범례 (few-shot):
- 구별 8례: 미숙함/가득/까먹는다/자랑/의도(수강신청)/부재(결혼기념일)/덜 쳐다봤다(비교어)/긍정 해부 — 각각 문제 문장→해부→사실 문장
- 질문 명중 3짝: 반복 인용→(오!!!?) / 정체→몸 감각 / 번아웃→쉬어가기

**원칙 안전장치 (변경 없음, 재확인)**:
- 인용은 원문 substring 서버 검증 유지 (지어내기 차단)
- 범주명·서사 명명("피해자 서사" 등) 사용자 노출 금지 — 해부 글의 명명은 운영자(사람)의 영역, 앱은 문장 지목까지만
- 개인 원문은 대본에 넣을 때 신원 단서 제거(이름·직장·교회 등), 짧은 조각만
- 톤: 해부 글의 단정문·건조함은 가져오되 "~입니다" → 앱 정본 "~이에요"체 유지
