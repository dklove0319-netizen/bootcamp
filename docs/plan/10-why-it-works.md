# 오제로가 작동하는 이유 — 수료자 후기 분석 (2026-07-05)

> 근거: 21·35일 레터 수료자 후기 원문 (운영자 제공). 후기 문장 ↔ 작동 기제 ↔ 앱 설계의 연결.
> 용도: 온보딩·랜딩 문구, 가격 방어 논리, 다음 층 기획의 근거 자료.

## 작동 기제 8개 — 후기가 직접 증언하는 것

### 1. 기술이 하나라서 자동화된다
- 후기: "망상은 망상이며, 그 자체로는 내 삶에 힘을 가질 수 없다는 사실이 제 안에서 확실해졌어요. 마음이 무슨 소리를 하든 갈 길을 가게 됐어요"
- 기제: 가르치는 기술이 사실/망상 구별 딱 하나. 단순한 기술 × 매일 반복 = 의식하지 않아도 돌아가는 자동 기술이 됨.
- 앱 연결: 하루 루프의 5단계(사용자 먼저 구별)가 이 반복 훈련의 본체. 카메라 검사 하나로 배움.

### 2. 남의 말이 아니라 자기 데이터가 스승이다
- 후기: "기록을 통해서 기분은 계속 오락가락하는 거지만, 나는 그냥 똑같은 사람 아닌가? 라는 생각을 처음 했어요. 기분과 나 자체는 별개라는 느낌이요!"
- 기제: 설득이 아니라 목격. 쌓인 자기 기록이 "기분≠나"의 증거가 됨 — 누가 말해줬으면 안 믿었을 것을 자기 데이터라서 믿는다.
- 앱 연결: 오늘의 눈금 추이 + 21일 격자 + 중간 거울 + 보고서 — 전부 "당신의 데이터를 당신에게 돌려주는" 장치.

### 3. 정체성을 준다 — 환자가 아니라 관찰자
- 후기: "관찰자라는 이름표의 명예를 지키기 위해 나약해지지 않겠습니다" / "함께 치열하게 싸워 온 전우이자 도반들"
- 기제: 사람은 프로그램은 그만둬도 정체성은 유지한다. '관찰자'라는 이름 + 익명 코드(oNNN)가 소속과 자부심을 만듦.
- 앱 연결: 관점 원칙(환자·내담자 아님) · observer_code · "관찰자" 호칭 일관 사용.

### 4. 보편성 — "나만 이상한 게 아니구나"
- 후기: "제가 '특별히 못난 사람'이 아니라 그냥 '평범한 사람'이 아닌가? (이게 많이 위로가 됐습니다)" / "다들 자신과의 싸움을 하고 계신 것 같아서요"
- 기제: 집단 치료 연구에서 가장 강력한 요인 중 하나(보편성). 타인의 진짜 기록을 살짝 보는 것만으로 자기 낙인이 풀림.
- 앱 연결: 동료 존재감 설계 — N명 카운트 + 익명 답변 1개. 피드가 아니라 "한 조각"으로 충분하다는 것을 후기가 증명 (잠깐 둘러본 것만으로 효과).

### 5. 회수의 기대 — 내 문장이 돌아온다
- 후기: "다음날 편지 열어볼 때마다 오늘은 내 얘기 없나 하고 기대하게 됨" / "메일에 언급이 되도 기쁘더라구요"
- 기제: 내가 쓴 것이 기억되고 돌아오는 경험 = 존재의 인정. 레터에서는 가끔·일부만 가능했던 것.
- 앱 연결: **앱의 구조적 우위** — 오프닝의 어제 문장 인용 + 원문 인용 질문이 매일, 전원에게, 개인화로 일어남. 레터가 손으로 하던 회수를 기계가 빠짐없이 한다.

### 6. 의식(ritual)이 된 시간
- 후기: "매일 8시 되기를 기다리다가... 화/금요일인 거 알고 살짝 서운했던 적도 있어요"
- 기제: 고정 시각이 하루의 닻이 됨. 기다림 자체가 참여의 증거.
- 앱 연결: 사용자 지정 고정 시각 + 저녁 푸시 1회. 레터는 주 2~5회였지만 앱은 매일 — 서운함의 빈자리를 채움.

### 7. 담백한 톤이 신뢰를 만든다
- 후기: "늘 담백한 텐션을 유지해주시는 것도 정말 너무 좋았습니다"
- 기제: 위로가 없으면 채점도 없다 — 과장 없는 톤이라서 안전하게 솔직해질 수 있음.
- 앱 연결: 톤 규칙(위로·감탄·이모지 금지) — 후기가 이 규칙의 시장 검증이다.

### 8. 끝이 있는 여정이라서 완주한다
- 후기: "지금 종료되는 게 좀 아쉬워요.. 뭐 더 프로그램 없을까요?" / "무의식탐구 다음 시리즈도 있다면 참여해보고 싶어요"
- 기제: 무한 서비스는 미루지만 끝이 있는 여정은 완주한다. 완결이 성취가 되고, 성취가 다음 단계 수요를 만든다.
- 앱 연결: 21일 구조 그 자체. 종료의 아쉬움 = 졸업 퍼널의 존재 증명 (PRD 로드맵).

## 후기의 개선 요청 ↔ 앱이 구조적으로 해결하는 것

| 후기 요청 | 앱에서의 답 |
|---|---|
| "3일 이상 메일 확인 안 하면 문자 하나 보내주는 것" | 이미 설계됨 — 3일 미기록 시 저녁 푸시 문구 변경 (지시서 5번). 수요 검증 완료 |
| 오타 · 잘못된 게시판 연동 · 질문 불일치 | 수작업 레터의 실수들 — 앱은 한 흐름 안에서 자동화라 구조적으로 사라짐 |
| "좀 더 이해하기 쉽게 설명" | 화면마다 한 단계씩 진행하는 루프 구조 + 첫날 안내 |
| "전날 답장에 대한 피드백이 더 상세하면" | **주의 — 의존 수요다.** 아래 참조 |

## 경계할 것: 의존 수요는 실재한다

"피드백을 더 상세히" 요청은 자연스럽지만, 이것이 바로 의존의 입구다 — 상세한 피드백일수록 사용자는 자기 구별 대신 AI/운영자의 판독을 기다리게 된다. 대응 원칙:
- 21일 거울 코스: 회수(인용)와 대조까지만. 해석 요청이 와도 늘리지 않는다.
- 35일 메스 코스(로드맵): 의도적으로 허가된 층에서만 수용 — 코스별 원칙 분리 (PRD 로드맵의 주의 항목).

## 졸업 퍼널 (전체 여정) → 상세는 PRD 로드맵

레터·콘텐츠 (무료 유입) → **21일 거울** (앱 입문) → **35일 메스** (심화) → **유지 모드 + 정기 재측정** (라이프스타일 층) → 졸업 관찰자의 공개 답변·후기가 다시 신규의 거울이 되는 순환.
