# 사용자 여정 검증 보고서 — 오제로의 거울

- 일자: 2026-07-08
- 방법: Playwright(사람 대신 브라우저를 눌러보는 로봇)로 로컬 서버(localhost:3000, 배포 커밋 df98d2e와 같은 코드)를 실제로 걸어봄. **AI는 전부 실호출** (측정 3회 · 대조 · 질문 · 제안 · 사흘의 거울 · 21일 보고서).
- 범위: 여정 9개, 체크 44건 (1차 32건 + 재검 12건). 시험 데이터는 o999(시험 관찰자)와 임시 발급 계정만 사용했고, 끝난 뒤 전부 삭제·카운터 원복.
- 스크린샷 17장 · 상세 로그: 세션 작업 폴더 `scratchpad/run1·run3` (임시 폴더 — 저장소에는 넣지 않음).

## 괜찮은 점 (근거와 함께)

**1. 판별 원칙이 실전에서 그대로 지켜진다 — 이 제품의 핵심이 작동한다.**
- "성의가 없다" → 망상, 이유: *"'성의'는 카메라에 안 담겨요 — 찍히는 건 오타 3개예요."* (구체성 기준 + 이유 예산 1곳만 사용)
- "죽고 싶다는 생각이 스쳤다" → **사실** (발생/내용 분리 규칙이 위기 문장에서도 정확히 적용됨)
- 깨끗한 사실 칸("지갑에 0원이 있다") → *"전부 카메라에 담기는 문장이에요."* — 칭찬 없이 조용한 통과
- 느낌 칸의 "무시당한 느낌이 올라왔다"를 판정하지 않고, 질문이 *"그때 몸에서는 무엇이 올라왔나요"* 로 그 아래 몸 신호를 물음 (자세규정의 느낌 칸 예외 규칙 그대로)

**2. 대조가 채점이 아니라 '나란히 놓기'다.** 일부러 거울과 어긋나게 구별해 봤더니 — "아무도 나를 도와주지 않는다"에 `내 구별: 사실 · 거울: 망상`이 점수·정답 표시 없이 나란히 놓이고, 이유는 *"'아무도'와 '도와주지 않는다'는 카메라에 담기지 않는 판정이에요."* 한 문장뿐.

**3. 위기 문장에도 루프가 멈추지 않는다.** "죽고 싶다는 생각이 스쳤다"를 쓰자 하단에 *"이야기할 수 있는 곳이 있어요 — 자살예방 상담전화 109 (24시간)"* 한 줄(닫기 가능)만 붙고, 기록·구별·질문은 그대로 진행됐다. 설계(조용한 B안) 그대로.

**4. 예외 상황이 전부 문장으로 받아진다.** 빈 기록 → "한 줄이라도 적어주세요." / 사실 칸 빈 채 → "사실 칸에 한 줄이라도 적어주세요." / 'abc' → "아이디 형식이 맞지 않아요. 예: o023" / 등록된 o999 이어받기 시도 → "이미 등록된 아이디예요." / 열쇠 없이 /me·/today·/report → 모두 "아직 오제로 아이디가 없어요" + 측정 입구 안내. 죽는 화면이 하나도 없었다.

**5. 한도 안내를 일 시키기 전에 한다.** 익명 2회차는 쓰기 시작 전에 *"오늘 측정은 여기까지예요. 내일 다시 비춰볼 수 있어요."* — 글을 다 쓰게 한 뒤 문을 닫는 흔한 실수를 피했다.

**6. 반복의 거울이 지어내지 않는다.** 사흘 기록에서 반복 인용이 전부 `2026-07-05 "또 무시당했다"` 형식(원문+날짜)이었고, 원문에 없는 문장은 0건 — 서버가 원문 대조로 걸러내는 설계가 실측으로 확인됨. 21일 보고서의 마지막 질문도 원문 인용 위에 섰다: *"나를 만만하게 보는 게 분명하다" → "그 '분명하다'는 판단이 서기까지, 상대의 말이나 표정에서 확인한 것은 무엇이었어요?"*

**7. AI 대기가 전부 5초 안이다.** 측정 판독 1.8~2.9초 · 거울 대조 4.5초 · 오늘의 질문 3.9초 · 행동 제안 3.0초 · 사흘의 거울 4.7초 · 21일 보고서 4.9초. 각 단계에 로딩 문구("비추고 있어요." 등)도 있다.

**8. 결제에 3중 방어가 있다.** 동의 없이 결제 → *"검증 동의에 체크해주세요"* 차단 → 동의 후 원화는 실제 토스 모래상자 결제창이 열림(`payment-gateway-sandbox.tosspayments.com` 확인) → 달러 테스트 승인 후 /me 에서 판매 카드가 사라지고 격자가 "1일차 · 21일"로 바뀌며 반복의 거울은 유지된다.

**9. 톤이 기계적으로도 깨끗하다.** 전 여정 화면 텍스트에서 느낌표·이모지·"축하/완벽/대단" 계열 어휘 0건 (자동 스캔). 영어권 접속 시 홈·측정이 영어로 자동 전환.

**10. 모바일(375px)에서 안 깨진다.** 홈·측정 모두 가로 넘침 없음, "자, 이제 이 기록에서 사실과 느낌, 행동만 구별해 봅니다" 제목이 정확히 한 줄.

## 개선점 (우선순위순, 근거와 제안)

**1. [지금 고쳐야 함] 신규 가입자가 o1000번대를 받는다.**
- 근거: 로봇이 실제로 "처음이에요"로 가입하니 **o1001**이 발급됐다. 번호 발급 로직이 "지금 등록된 번호 중 최댓값 + 1"인데, 시험 관찰자 **o999가 최댓값**이라 그 다음은 항상 o1000대가 된다. 현재 실사용자 o056~o059가 이미 있어서, 다음 실가입자부터 바로 겪는 **라이브 문제**다.
- 제안: 번호 발급에서 900번대 이상(시험 대역)을 제외하거나, 시험 관찰자를 번호 체계 밖 코드(예: test)로 옮기기. (이번 검증에서 생긴 o1000·o1001은 삭제해 원상 복구해 둠.)

**2. [공개 확장 전] 같은 와이파이를 쓰면 한도를 나눠 갖는다.**
- 근거: 횟수 제한이 순수하게 접속 주소(IP) 지문으로만 계산된다 (코드 확인 + 실측). 카페·회사 공용 와이파이에서는 다른 사람이 먼저 1회를 쓰면 내 첫 체험이 막힌다. 아이디 보유자(3회)도 같은 IP끼리 3회를 공유한다.
- 제안: 오제로 아이디 보유자는 IP 대신 비밀 열쇠 기준으로 세기. 익명은 지금대로 IP 기준 유지 (남용 방지).

**3. [공개 확장 전] 한도에 걸린 사람에게 쓰기 화면이 0.5초쯤 먼저 보인다.**
- 근거: 입구 한도 확인이 화면을 띄운 뒤에 도착해서, 차단 대상자도 쓰기 화면을 잠깐 보고 나서 차단 화면으로 바뀐다. 자동화 로봇이 이 틈에 두 번 걸렸다 — 빠르게 쓰기 시작하는 실사용자도 걸릴 수 있는 창이다.
- 제안: 확인이 끝날 때까지 "거울에 비춰보기" 버튼만 잠그면 화면 깜빡임 없이 해결된다.

**4. [메모] 결제 여정에서 콘솔 400 두 건이 1회 관찰됐다.**
- 근거: 1차 실행의 토스 결제창 구간에서만 나타났고, 주소까지 기록한 재검에서는 재현되지 않았다. 우리 앱 창구(결제 승인·내 거울·오늘·반복의 거울)는 전수 정상(200) 확인. 토스 SDK가 부르는 외부 리소스로 추정 — 실결제 키 전환 시험 때 한 번 더 보면 된다.

**5. [이미 계획됨 — 리마인드] /seed 검증 도구가 라이브에 열려 있다.** 등록된 열쇠가 있어야 작동해 남용은 어렵지만, 공개 확장 전 제거 대상(plan.md 스코프컷)이다.
